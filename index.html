Core concepts
📋

Jobs

A sequence of module actions defined in a portable .job.json case file. Shareable, versionable, replayable. The agent's unit of work.

🧩

Modules

Pluggable wrappers around API surfaces. Built-ins handle flow primitives. Bring your own for private APIs — packaged as .dpmod.zip bundles.

📄

SKILL.md

The agent briefing. A structured contract that tells any AI agent how to operate your setup: happy path, troubleshooting ladder, command surface. Drop it in context. It knows what to do.

🗂️

Artifacts

Every run writes a full record: requests, responses, events, module resolution. Assert offline. Replay later. Hand it to your agent as context without a network call.

The briefing format

Your agent reads
SKILL.md first.

Every Dispatch setup ships with a SKILL.md — a structured document your AI agent reads before acting.

It defines the preflight sequence, the happy path, the troubleshooting ladder. The agent doesn't explore. It follows the brief.

Write your own for private modules. The format is open. The contract is yours.

📄 SKILL.md
---
name: my-api-dispatch
description: API workflows via dispatch
---

## Preflight
1. dispatch doctor --no-network
2. dispatch self-check

## Default Flow
1. dispatch job validate --case <path>
2. dispatch job run --case <path>
3. dispatch job assert --run-id latest

## Troubleshooting
1. dispatch job inspect --run-id latest
2. dispatch job readable --run-id latest
